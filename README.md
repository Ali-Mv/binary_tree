# باغبانی به سبک باینری (درخت باینری کامل)

این پروژه تمرینی برای پیاده‌سازی ساختمان داده‌ی **درخت باینری کامل (Complete Binary Tree)** در درس برنامه‌سازی پیشرفته است. هدف اصلی، تمرین مفاهیم پیشرفته ++C مانند قالب‌ها (Templates)، مدیریت استثناها و چندریختی است.

## ویژگی‌های اصلی پروژه

- **Complete Binary Tree:** حفظ ساختار درخت به گونه‌ای که گره‌ها همیشه از چپ به راست در آخرین ردیف پر می‌شوند.
- **Generic Programming:** استفاده از `Template` برای پشتیبانی از انواع داده‌های مختلف (`int`, `double`, شیء و...).
- **Polymorphism:** پیاده‌سازی کلاس پایه انتزاعی و متدهای مجازی خالص (`Pure Virtual Functions`).
- **Memory Management:** مدیریت دستی حافظه (پویا) و پیاده‌سازی **Deep Copy** (سازنده کپی و اپراتور انتساب) بدون استفاده از STL.
- **Exception Handling:** مدیریت خطاهای زمان اجرا (مانند حذف از درخت خالی) با طراحی کلاس استثنای سفارشی.

## ساختار کلاس‌ها

### ۱. مدیریت خطا
- `TreeException`: ارث‌بری از `std::exception` برای مدیریت خطاهای اختصاصی درخت.

### ۲. گره‌های درخت
- `BaseNode<T>` (کلاس انتزاعی): شامل دیتای گره و متد مجازی خالص `printInfo`.
- `TreeNode<T>`: ارث‌بری از کلاس پایه و اضافه کردن اشاره‌گرهای `left` ،`right` و `parent`.

### ۳. مدیریت درخت
- `CompleteBinaryTree<T>`: کلاس اصلی برای مدیریت عملیات‌های درخت:
    - `insert`: درج گره در اولین موقعیت خالی (حفظ ویژگی کامل بودن).
    - `removeLast`: حذف آخرین گره اضافه شده به درخت.
    - `levelOrderTraversal`: پیمایش سطح‌به‌سطح (BFS) برای نمایش ترتیب گره‌ها.
    - `clear`: پاک‌سازی کامل حافظه و جلوگیری از نشت حافظه (Memory Leak).

## محدودیت‌های پروژه

- مجاز نبودن استفاده از کتابخانه‌های آماده (STL) مانند `vector` یا `queue`.
- پیاده‌سازی تمامی ساختارها به صورت بازگشتی یا با مدیریت مستقیم اشاره‌گرها.
- استفاده از ساختار چند فایلی (Multi-file) برای سازمان‌دهی کدها.
